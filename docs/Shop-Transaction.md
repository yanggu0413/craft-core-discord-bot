# 📖 買家交易與預約

本章節介紹非商店擁有者（顧客）如何與箱子商店進行互動，包含購買、出售以及當商品缺貨時的預約流程。

---

## 1. 🔍 啟動交易對話
顧客對著任何商店箱子按**【左鍵】**，聊天欄將會跳出精美的商店詳情面板（該面板只有該顧客看得見）：

```text
§6=================== 商店交易 ===================
§f商品名稱: 鑽石
§f商店主人: im_little_rory
§a[出售商品] -> 價格: §e$100.0§f (庫存: 64)
§b[收購商品] -> 收購價: §e$50.0§f (可收購空間: 1728)
§6---------------------------------------------
§e★ 請選擇您的操作：
§f- 請在聊天欄輸入「§a買§f」開始向商店購買商品。
§f- 請在聊天欄輸入「§b賣§f」將背包中的物品賣給商店。
§f- 輸入「§c取消§f」結束交易。
§6============================================
```

顧客此時會進入 **30 秒交易狀態**。玩家發送的下一條訊息將會被系統攔截（不會廣播給伺服器其他玩家），並解析為交易類型。超時無響應或輸入「取消」將會自動退出交易。

---

## 2. 🛒 購買商品 (顧客買，商店賣)
1. 在聊天欄輸入 **`買`** 或是 **`buy`**。
2. 系統提示：
   `§b[Craft-Core] §f您已選擇【購買】。請在聊天欄輸入欲購買的「數量」（如: 64），或輸入「取消」取消：`
3. 輸入正整數數量（例如 `32`），系統將會進行檢查：
   - 檢查顧客帳戶餘額是否足夠。
   - 檢查商店箱子內該商品的實際庫存是否足夠。
   - 檢查顧客背包空間是否放得下。
4. 檢查通過後扣除顧客金幣，將商品轉移至顧客背包，並播放 `entity.experience_orb.pickup` 叮的一聲音效！交易金額存入商店的累積營收。

---

## 3. 💰 出售商品 (顧客賣，商店收購)
1. 在聊天欄輸入 **`賣`** 或是 **`sell`**。
2. 系統提示：
   `§b[Craft-Core] §f您已選擇【出售】。請在聊天欄輸入欲出售的「數量」（如: 32），或輸入「取消」取消：`
3. 輸入正整數數量（例如 `64`），系統將會進行檢查：
   - 檢查顧客背包內是否擁有足夠的該商品。
   - 檢查商店主人帳戶的金幣餘額是否足夠支付。
   - 檢查商店箱子內剩餘的槽位（Slot Space）是否放得下要出售的物品。
4. 檢查通過後，從顧客背包扣除商品，存入商店箱子，並從商店主人帳戶餘額中扣除相應金額支付給顧客，播放叮的一聲音效！

---

## 4. 📝 缺貨預約訂單 (Out-of-Stock Reservations)
若商店目前的庫存為 `0`，但該商店開啟了「出售」功能，買家仍可以發起預約委託。

### 預約流程：
1. 顧客左鍵點擊箱子並輸入 `買`。
2. 系統偵測到庫存不足，但會提示：
   `§b[Craft-Core] §e該商品目前缺貨。您是否要付清款項以預約此商品？`
   `§f- 輸入您要預約的「數量」進行預付款預約，或輸入「取消」退出。`
3. 輸入數量後，系統會即時扣除顧客的金幣並託管（Escrow）在該商店的預約單中。

### 自動委託發貨：
*   當商店主人隨後對該箱子進行**補貨**時，系統會自動在後台依據預約的先後順序進行配貨。
*   物品會自動從箱子扣除，並透過**離線快遞信箱**發送給買家（即使買家離線也會在下次上線時收到）。同時，託管的金幣將正式發放給商店主人。

### 預約取消與退款：
*   買家若不想繼續等待，可以隨時輸入 `/shop reservations` 查看自己當前所有的掛單預約。
*   點選取消按鈕，託管的預付款金幣將會 **100% 全額退還** 給買家，並註銷該筆預約單。
